# Evidencias para la EFT

Antes de entregar, agregar capturas actuales en una carpeta llamada `evidencias-eft/`.

## Capturas obligatorias

- [ ] Repositorio GitHub en la rama `main`.
- [ ] Lista de ramas.
- [ ] Rama `feature/evidencias-eft`.
- [ ] Pull Request hacia `main`.
- [ ] GitHub Actions ejecutado.
- [ ] Jobs del pipeline en verde.
- [ ] `Dockerfile`.
- [ ] `docker-compose.yml`.
- [ ] `docker compose ps` con contenedores arriba.
- [ ] Endpoint `http://localhost:8000/health`.
- [ ] Endpoint `http://localhost:8000/metrics`.
- [ ] Prometheus en `Status > Targets` con el microservicio `UP`.
- [ ] Grafana mostrando dashboard.
- [ ] SonarCloud o SonarQube.
- [ ] Snyk, Trivy o Dependabot.

## Capturas que ayudan mucho

- [ ] Commits recientes con mensajes claros.
- [ ] Secrets configurados, ocultando los valores.
- [ ] Evidencia de un pipeline fallido por prueba o seguridad.
- [ ] Evidencia del pipeline corregido y exitoso.

## Recomendacion

No usar capturas antiguas si fueron observadas en EP3. Para la EFT conviene tomar capturas nuevas y explicar cada una durante la presentacion.
